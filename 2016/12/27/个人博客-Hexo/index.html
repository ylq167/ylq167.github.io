个人博客 Hexo

  2016-12-27

终于折腾好了Hexo

心心念念一年多的个人独立博客，今天花了一下午终于搭好了。采用的是 Hexo 的框架，托管在 Github Pages 上，经过 DNS 解析，现在可以通过去年买的域名 11167.xyz 来访问。本篇描述了个人独立博客的由来与搭建过程。

为什么要建立个人独立博客

个人独立博客，一般指在空间、域名和内容上相对独立的博客。
一直希望有一个地方能记录工作生活的点滴和思考。总看到别人说「请移步至我的个人博客查看详细」会觉得很厉害。相对于市面上现有的各种平台，比如简书，微信公众号，知乎专栏和各博客平台等，通过自己的专属域名访问个人独立博客，有一种开辟了一块空间，而不是寄人篱下的感觉。在互联网行业，求职信中附上自己的个人博客也是加分项之一。下面根据我的理解说说个人独立博客的优缺点。

优点

空间独立，是互联网空间中完全属于自己的一部分。
内容自由，不用经过审查，没有平台限制。
形式多样，多种框架模版可以选择，更可以通过代码创造无数的可能。
技术成熟，已经有很多博客框架可以选择，无需编程基础，花一定的时间就可以上手。

缺点

需要自己购买域名。
有一定的技术门槛。
更新维护不如现有平台方便。
所以，如果你也想通过个人独立博客来记录和展示，就请立即行动起来吧。

为什么选择 Hexo

无需从零开始，大牛们已经建立了很多个人独立博客的框架，包括 WordPress、Jekyll、以及我所使用的 Hexo 等。

选择 Hexo 的最主要的原因是快速、简洁且高效。当然 Hexo 只是静态博客框架，无法实现复杂的数据库功能。也许之后还会尝试 Jekyll ，但目前的目标就是把 Hexo 掌握熟练。

关于博客框架的选择可以通过搜索了解更多的信息，如果时间充沛也可以对感兴趣的逐一尝试。

为什么托管在 GitHub Pages

首先介绍 GitHub Pages 是什么，简单来说 GitHub Pages 原本用于介绍托管在 GitHub 的项目。它提供了 300Mb 的免费空间，所以越来越多的人在 GitHub Pages 中搭建一个个人独立博客。这样就省去了自己购买服务器的麻烦，对大多数人的静态博客来说，空间是够用的。稳定性有非常好。同时，GitHub 作为全球程序员大牛的聚集地，本身用来管理项目，又有很多开源项目。IT 行业的人理应对其有更多的了解和接触。

开始动手吧，只需要一台可以接入互联网的个人电脑，立即开创属于自己的空间。
